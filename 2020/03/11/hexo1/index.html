Step0.乱七八糟的介绍

Hexo博客的工作流程如下，用户使用MarkDown撰写文章，通过hexo生成静态网页，再通过GithubPages部署后展示给互联网上的访客。

优点：轻量化免费好操作

缺点：过于依赖本地环境，更换本地环境就极为不方便

Step1.环境搭建

因为hexo自身基于Node.js，而后需要git bush对github端的仓库进行操作，所以我们需要先安装Node.js和Git

安装包下载地址

Git：https://git-scm.com/downloads
Node.js：https://nodejs.org/zh-cn

安装过程一路默认Next就好。

安装后的校验

在CMD中输入

git –version
node -v
npm -v

得到如下的返回值就说明安装成功了~

Step.2 创建SSH密匙与Github链接

由于我们使用Github Pages功能来承载我们的博客，所以我们需要用一个SHH密匙让我们的本地环境和Github连接起来。

1.创建本地文件夹与SSH密匙

新建一个准备储存Hexo本地文件的文件夹，在文件夹内右键->GitBush，输入如下指令。

git config –global user.name “你的Github用户名”
git config –global user.name “你的Github邮箱”
ssh-keygen -t rsa -C “你的Github 邮箱”
然后一路默认回车

2.将密匙添加进Github中

在C:\Users\你的用户名.ssh地址中找到 id_rsa.pub 用记事本打开并复制其中内容。
注意一定要复制id_rsa.pub里面的，不要复制旁边那个不带扩展名的。

Github->头像->Setting->SSH and GPG keys->New SSH key,粘贴到Key框中点击下方绿色按钮就OJBK了。

3.验证连接状态

在GitBash中输入：

ssh -T git@github.com

出现如下返回即是连接成功

4.创建仓库

仓库名一定是“你的用户名.github.io” 不要起成博客的名字了，会404

Step3.在本地安装hexo

1.npm安装Hexo主程序

在你的博客文件夹下，右键->GitBush
输入如下命令：

npm install -g hexo-cli

输入完之后会没有反应一阵子，耐心等等出现一大堆字母之后就可以了。

2.初始化Hexo及本地预览

依次输入：

hexo init 初始化
npm install 安装组件

完成后输入

hexo g
这是生成页面的指令，以后会经常用到
hexo s
这是本地预览指令

之后访问 >http://localhost:4000 出现hexo默认主题界面就是安装成功了！
